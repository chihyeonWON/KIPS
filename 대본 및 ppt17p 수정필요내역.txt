웹 크롤링 
java기반 jsoup라이브러리를 이용해서 크롤링
데이터 테이블에 맞게 크롤링 과정에서 자료형 변환과정 진행

데이터 베이스 
크롤링 데이터를 바탕으로 데이터 베이스 테이블 설계 및 구현 
위치데이터를 기반으로 시간과 기상 데이터를 유지 할 수 있도록 함
aws에서 제공하는 rds를 이용하여 관계형 데이터베이스 구축
트리거를 이용하여 수집한 데이터가 있다면 갱신, 없다면 삽입을 진행하도록 제작

웹 서버 및 옷추천 리스트
mysql을 통해 데이터베이스에 데이터 저장을 하도록 연결 
들어온 요일과 날씨에 관한 데이터에 없다면 모든 데이터를 저장하지만 
이미 데이터가 있으면, 크롤링을 다시하여 변경사항만 갱신하도록 설계
수정 필요 -ppt17- 
설명이 db와 어플리케이션간의 데이터 송수신 확인으로 적혀있는데
정확하게는 옷추천db와 날씨 db간의 데이터 송수신이 작동하는지 postman을 통한 확인과정입니다.

어플리케이션
플로팅 버튼 클릭을 통해 위경도값을 받도록 설계
받아온 위경도 값을 geocoder클래스를 이용하여 실제 주소로 변환 
이때 주소는 시,군,동(원주시 흥업면)으로 3개의 변수로 만든다.

통신의 경우 retrofit2을 통해 진행하며, 데이터 송수신에는 Gson을 선언하여
json파일을 사용 할 수 있도록 한다.

날씨 그래프의 경우 mpAndroidchart 라이브러리(오픈소스)를 사용하여 제작하였다.
